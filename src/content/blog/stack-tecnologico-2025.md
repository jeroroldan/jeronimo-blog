# 🏗️ Stack Tecnológico Recomendado 2025

> **Nota:** La elección del stack siempre debe adaptarse a los recursos humanos, limitaciones y requerimientos específicos de cada proyecto.

## 📚 Índice
1. [Frontend](#frontend)
2. [Backend](#backend)
3. [Infraestructura y DevOps](#infraestructura)

---

## 🎨 Frontend {#frontend}

- **Rspack:** Bundler ultrarrápido y compatible, ideal para acelerar builds y desarrollo.
- **Astro:** Framework principal para sitios web estáticos y dinámicos. Permite integrar:
    - **React** o **Angular 19** para componentes interactivos o renderizado incremental.
- **Tailwind CSS v4:** Para proyectos personales por su rapidez y productividad.
- **CSS Modules:** Para proyectos que requieren mayor control y evitar dependencias adicionales.
- **Three.js:** Para modelado y visualización 3D en el navegador.
- **Biome:** Herramienta de linting y formateo para mantener la calidad del código.

---

## 🖥️ Backend {#backend}

- **Go:** Lenguaje principal por su rendimiento, concurrencia y minimalismo.
    - **Echo:** Framework web en Go, enfocado en sincronía y simplicidad.
- **MongoDB:** Base de datos NoSQL, usando el driver oficial de Go.
- **Colly:** Librería para scraping web eficiente en Go.
- **Kiddy (kdb):** Alternativa a Redis para caché, simple y rápida.

---

## ☁️ Infraestructura y DevOps {#infraestructura}

- **Supabase:** Alternativa open source a Firebase para autenticación, base de datos y almacenamiento.
- **Vercel** o **Netlify:** Para despliegue de frontend, con entornos de preview colaborativos.
- **Docker:** Contenerización y despliegue en servidores VPS para mayor control y portabilidad.
- **Playwright:** Pruebas end-to-end automatizadas para asegurar calidad en el flujo completo.

---

> **Resumen:** Este stack prioriza velocidad, control, facilidad de integración y herramientas modernas, pero siempre debe adaptarse a tu equipo y contexto.
