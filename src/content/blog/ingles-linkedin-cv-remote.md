# Inglés para CV, LinkedIn y Aplicar a Trabajos Remotos

Conseguir trabajo en Estados Unidos o en empresas que contratan remoto desde Latinoamérica no depende solo de programar bien. También depende de saber presentar tu experiencia en inglés.

Un buen CV en inglés no es una traducción literal de tu CV en español. Es un documento comercial: vende tu valor, muestra resultados y está escrito para que un reclutador entienda rápido qué podés aportar.

Esta guía te va a ayudar a escribir un CV profesional, optimizar tu perfil de LinkedIn, redactar mensajes para aplicar a trabajos remotos y mejorar tu comunicación durante el proceso de aplicación.

---

## Objetivo de esta guía

Al terminar esta guía vas a poder:

- Escribir un CV en inglés orientado a empresas de Estados Unidos.
- Convertir responsabilidades en logros.
- Usar verbos de acción profesionales.
- Optimizar tu perfil de LinkedIn en inglés.
- Escribir cover letters cortas y efectivas.
- Enviar mensajes de networking a reclutadores y hiring managers.
- Aplicar a trabajos remotos con más claridad.
- Evitar errores comunes que hacen que tu aplicación se vea poco profesional.

---

## Diferencia entre CV y resume

En Estados Unidos, para trabajos normales se usa principalmente un **resume**, no un CV académico.

### Resume

- 1 página, a veces 2 si tenés mucha experiencia.
- Enfocado en resultados y habilidades relevantes.
- Se adapta a cada puesto.
- No incluye foto, edad, estado civil ni dirección completa.

### CV

- Más largo.
- Usado en ámbitos académicos, médicos o científicos.
- Incluye publicaciones, investigación, conferencias y educación detallada.

Para aplicar a trabajos de software, diseño, marketing, soporte, operaciones o roles técnicos, normalmente necesitás un resume.

## Estructura recomendada de un resume en inglés

Una estructura sólida para roles técnicos:

```
Name
City, Country | Email | LinkedIn | GitHub | Portfolio

Professional Summary
Experience
Projects
Skills
Education
Certifications
```

Si tenés poca experiencia, podés poner proyectos antes que experiencia. Si tenés experiencia profesional fuerte, poné experiencia antes que proyectos.

## Professional Summary

El resumen profesional va al inicio. Tiene que ser breve: 2 a 4 líneas.

### Fórmula

```
[Role] with [years] of experience in [area].
Experienced with [technologies/tools].
Strong background in [type of problems].
Interested in [goal related to role].
```

### Ejemplo backend

> Backend Developer with 3+ years of experience building REST APIs, database systems and cloud-based applications. Experienced with Node.js, TypeScript, PostgreSQL, Docker and AWS. Strong background in performance optimization, authentication and clean API design.

### Ejemplo frontend

> Frontend Developer with experience building responsive web applications using React, TypeScript, CSS and modern UI libraries. Strong focus on performance, accessibility and user experience. Comfortable working in agile teams and collaborating with designers and backend engineers.

### Ejemplo junior

> Junior Software Developer with hands-on experience building web applications with JavaScript, React and Node.js. Passionate about clean code, problem-solving and continuous learning. Built personal and team projects involving authentication, REST APIs and responsive interfaces.

## Experience: la sección más importante

La mayoría de los resumes fallan porque solo listan tareas.

### Débil

```
Worked on backend APIs.
Fixed bugs.
Used React.
Talked with the team.
```

### Fuerte

```
Built REST APIs for a customer management system using Node.js and PostgreSQL.
Reduced API response time by 35% by optimizing database queries and adding indexes.
Collaborated with frontend developers to design API contracts and improve error handling.
```

La diferencia es clara: la segunda versión muestra acción, tecnología y resultado.

## Fórmula para escribir bullets profesionales

Usá esta fórmula:

```
Action verb + what you did + technology/context + result
```

```
Improved frontend performance by reducing bundle size and lazy-loading heavy components.
Automated deployment tasks with GitHub Actions, reducing manual release steps by 60%.
```

## Cómo escribir experiencia si no tenés números

No todos los trabajos tienen métricas. Si no tenés números, usá impacto cualitativo.

### Sin métrica

```
Built a reporting dashboard for the operations team.
```

### Mejor

```
Built a reporting dashboard for the operations team, making it easier to track daily metrics and identify bottlenecks.
```

```
Built a reporting dashboard for the operations team, reducing manual reporting time from 2 hours to 20 minutes per day.
```

## Skills: cómo organizarlas

No pongas una lista gigante sin estructura. Agrupá tus habilidades.

```
Languages: JavaScript, TypeScript, Python
Frontend: React, Next.js, HTML, CSS, Tailwind CSS
Backend: Node.js, Express, REST APIs, GraphQL
Databases: PostgreSQL, MySQL, MongoDB
Tools: Git, Docker, GitHub Actions, AWS, Linux

```

### Evitá

```

```

Eso mezcla tecnologías con habilidades blandas y se ve desordenado.

## Cover letter en inglés

Una cover letter no tiene que ser larga. Para roles técnicos, muchas veces no la leen completa. Usá 3 a 4 párrafos breves.

### Estructura

```
1. Saludo
2. Por qué te interesa el rol
3. Qué experiencia relevante traés
4. Cierre profesional
```

### Plantilla

```
Hi [Hiring Manager or Team],

I'm excited to apply for the [Role] position at [Company]. I'm interested in this opportunity because [specific reason related to company/product/mission].

I have experience with [relevant technologies] and I've worked on [relevant type of projects]. In my previous role, I [specific achievement].

I believe my background in [skill 1], [skill 2] and [skill 3] would help me contribute to your team. I would be glad to discuss how my experience aligns with your needs.

Thank you for your time and consideration.

Best regards,
[Your Name]
```

### Ejemplo

```
Hi Hiring Team,

I'm excited to apply for the Backend Developer position at your company. I'm interested in this opportunity because I enjoy building reliable APIs and scalable systems for real users.

I have experience with Node.js, TypeScript, PostgreSQL and Docker. In my previous role, I developed REST APIs for a customer management platform and improved response time by optimizing database queries.

I believe my background in backend development, clean API design and problem-solving would help me contribute to your engineering team. I would be glad to discuss how my experience aligns with your needs.

Thank you for your time and consideration.

Best regards,
Jerónimo
```

## About section para LinkedIn

Tu About no tiene que ser una biografía larga. Tiene que responder rápido:

- Qué hacés.
- Con qué tecnologías trabajás.
- Qué problemas resolvés.
- Qué tipo de oportunidades buscás.

### Plantilla

```
I'm a [role] with experience in [area].
I work with [technologies] to build [type of products].
I enjoy solving problems related to [problem 1], [problem 2] and [problem 3].
Recently, I worked on [brief project or achievement].
I'm interested in remote opportunities where I can contribute to [goal].
```

### Ejemplo

```
I'm a Backend Developer with experience building REST APIs, database systems and cloud-based applications. I work with Node.js, TypeScript, PostgreSQL and Docker to create reliable services for web and mobile products.

I enjoy solving problems related to API design, performance, authentication and clean architecture. Recently, I worked on improving backend response times and documenting API contracts for frontend integration.

I'm interested in remote opportunities where I can contribute to high-quality software, learn from strong engineering teams and keep growing as a developer.
```

## Aplicar a trabajos remotos

Para trabajar remoto desde Latinoamérica para empresas de Estados Unidos, necesitás comunicar confianza en tres áreas:

```
Comunicación
Confiabilidad
Overlap horario
```

### Frases útiles para tu perfil

```
Comfortable working remotely.
Experienced with async communication.
Available for overlap with US time zones.
Strong written communication skills.
Used to documenting progress and blockers.
```

---

## Cómo mencionar disponibilidad horaria

Si el trabajo requiere overlap con Estados Unidos, sé claro.

```
I'm available for 4 hours of overlap with US Eastern Time.
I'm comfortable working during US business hours.
I can attend meetings between 9 AM and 1 PM EST.
```

Evitá sonar ambiguo:

```
I can work anytime.
```

Eso puede parecer poco realista.

---

## Cómo explicar que sos de otro país

No hace falta esconderlo. Presentalo con claridad.

```
I'm based in Argentina and I'm looking for remote opportunities with US-based teams.
I have experience working with distributed teams across time zones.
I'm comfortable using async communication tools like Slack, Notion and Jira.
```

## Cómo adaptar tu resume a cada puesto

No mandes el mismo resume a todos lados.

### Antes de aplicar

1. Leé la descripción del puesto.
2. Marcá tecnologías y responsabilidades clave.
3. Cambiá el resumen profesional.
4. Reordená bullets según relevancia.
5. Ajustá skills.
6. Usá las mismas palabras que aparecen en el job posting.

### Ejemplo

Si el puesto dice:

```
Experience with REST APIs, PostgreSQL, testing and AWS.
```

Tu resume debería incluir:

```
Developed REST APIs with PostgreSQL and wrote tests for critical endpoints.
Deployed services using AWS infrastructure.
```

## ATS: Applicant Tracking Systems

Muchas empresas usan sistemas automáticos para filtrar resumes.

### Tips básicos

```
Usá formato simple.
Evitá columnas complejas.
No uses imágenes dentro del resume.
Usá palabras clave del job posting.
Incluí tecnologías exactas: PostgreSQL, React, TypeScript, AWS.
Guardá el archivo como PDF.
```

### Nombre del archivo

```
Jeronimo-Vivas-Backend-Developer.pdf
Jeronimo-Vivas-Full-Stack-Developer.pdf
```

Evitá:

```
resume_final_final_2.pdf
cv_nuevo_actualizado.pdf
```

## Errores comunes en CVs en inglés

### Error 1: Traducción literal

```
I was in charge of making APIs.
```

Mejor:

```
I developed REST APIs.
```

---

### Error 2: Usar pronombres personales

En resumes en inglés se omiten "I" y "my".

```
I developed APIs.
```

Mejor:

```
Developed REST APIs.
```

---

### Error 3: Poner nivel de inglés poco profesional

Evitá:

```
English: conversational
English: medium
English: basic-intermediate
```

Mejor:

```
English: Professional working proficiency
```

Si tu nivel es bajo, mejor no lo destaques en el resume. Preparalo antes de entrevistas.

---

### Error 4: Poner demasiadas tecnologías

No pongas todo lo que alguna vez viste.

```
JavaScript, React, Angular, Vue, Node, Laravel, Django, Flutter, AWS, Azure, GCP, Kubernetes, Terraform, C, C++, Java, PHP
```

Eso puede generar preguntas que no podés responder.

---

### Error 5: No revisar gramática

Errores comunes:

```
I work with React since 2022.
```

Correcto:

```
I have worked with React since 2022.
```

```
I did a project where I builded an API.
```

Correcto:

```
I built an API in a project focused on task management.
```

### Paso 1: Elegí un puesto real


### Paso 3: Reescribí tu resumen

```

```

### Paso 4: Reescribí 3 bullets

Convertí tareas en logros.

## Recordatorio final

Tu CV y LinkedIn no tienen que mostrar que sabés todo. Tienen que mostrar que podés resolver problemas relevantes para el puesto.

La fórmula es simple:

```
Claridad + relevancia + resultados + comunicación profesional
```

Si tu perfil comunica eso en inglés, vas a destacar mucho más que alguien que solo traduce su experiencia palabra por palabra.
