# Inglés para Onboarding y Primer Día de Trabajo en Estados Unidos

Conseguir el trabajo es solo una parte. El primer desafío real empieza cuando entrás al equipo.

El onboarding en Estados Unidos suele ser muy estructurado: reuniones con HR, configuración de herramientas, presentaciones con el manager, acceso a sistemas, documentación, reuniones con compañeros y objetivos para las primeras semanas. Si no estás preparado, podés sentir que todo va demasiado rápido.

Esta guía te da frases, vocabulario y estrategias para manejar tu primer día y primeras semanas en inglés profesional.

---

## Objetivo de esta guía

Al terminar esta guía vas a poder:

- Presentarte en tu primer día.
- Entender reuniones de onboarding.
- Pedir aclaraciones sin sonar inseguro.
- Manejar setup técnico y accesos.
- Hablar con HR sobre beneficios, payroll y documentos.
- Participar en daily meetings.
- Tomar notas y confirmar entendimiento.
- Comunicar blockers de forma profesional.
- Crear buena impresión en las primeras semanas.

---

## Mentalidad para el primer día

No se espera que sepas todo el primer día.

Lo que sí se espera es que seas:

```
Puntual
Curioso
Respetuoso
Organizado
Proactivo
Claro al comunicar dudas
```

Tu trabajo no es fingir que entendés todo. Tu trabajo es aprender rápido y comunicar bien.

## Frases para presentarte

### En una reunión con el equipo

```
Hi everyone, I'm [Name].
I'm excited to join the team.
I'm based in [Country].
I'll be working as a [Role].
I'm looking forward to learning from all of you.
```

### Ejemplo

> Hi everyone, I'm Jerónimo. I'm excited to join the team. I'm based in Argentina and I'll be working as a backend developer. I'm looking forward to learning from all of you and contributing to the project.

### Más casual

```
Hey everyone, I'm [Name].
I'm the new [Role] on the team.
Happy to be here.
Looking forward to working with you.
```

## Primer día: frases esenciales

### Si no entendés algo

```
Could you repeat that, please?
Could you say that again a bit slower?
I didn't catch that.
Sorry, I missed the last part.
Could you clarify what you mean by [term]?
```

### Si necesitás tiempo

```
Let me take a note of that.
I'll review the documentation and come back with questions.
I want to make sure I understand this correctly.
Let me double-check that.
```

### Si entendiste

```
That makes sense.
Got it, thanks.
Understood.
Thanks, that clarifies it.
I think I have it now.
```

## Cómo pedir ayuda sin sonar débil

Pedir ayuda es normal. La clave es mostrar que ya intentaste resolverlo.

### Fórmula profesional

```
I tried [action], but I'm still blocked on [issue].
Could you help me understand [specific point]?
```

### Ejemplo

> I tried following the setup guide and installing the dependencies, but I'm still blocked on the database connection. Could you help me understand which environment variables I should use locally?

### Más simple

> I read the documentation, but I'm not sure how to configure the API key. Could you point me to the right place?

En muchas empresas, HR te va a explicar beneficios y paperwork.

### Frases útiles

```

How does PTO work?
Is there a waiting period?
```

```
Could you send me a link with the details?
Is there documentation I should review?
```

## Setup técnico

El setup técnico puede ser confuso porque mezclan inglés técnico, nombres de herramientas y procesos internos.

### Frases para pedir acceso

```
I don't have access to the repository yet.
Could you add me to the GitHub organization?
I need access to the staging environment.
Could you invite me to the Slack workspace?
I don't see the Jira board yet.
Could you check my permissions for the database?
```

### Frases para setup local

```
I'm setting up the project locally.
I cloned the repository.
I installed the dependencies.
The build is failing.
The tests are failing.
I'm getting an error when I run the app.
The environment variables are not configured.
The API key seems to be missing.
```

### Ejemplo de mensaje en Slack

```
Hi team, I'm setting up the project locally. I cloned the repo and installed the dependencies, but the app fails when I run npm start. The error says the database URL is missing. Could someone point me to the local setup documentation?
```

## Daily standup en inglés

El daily suele ser breve. Cada persona responde tres cosas:

```
What did I do yesterday?
What will I do today?
Do I have any blockers?
```

### Plantilla

```
Yesterday, I worked on [task].
Today, I'll focus on [task].
I'm currently blocked by [blocker].
```

### Ejemplo

> Yesterday, I worked on setting up the local environment and reading the API documentation. Today, I'll focus on running the tests and understanding the main user flow. I'm currently blocked by access to the staging environment.

### Si no hiciste mucho

```
Yesterday was mostly onboarding and setup.
Today, I'll continue reviewing the documentation and setting up the project.
No blockers for now.
```

## Cómo confirmar entendimiento

No digas solo "okay" si necesitás precisión.

### Frases profesionales

```
Just to confirm, I should [action].
So the next step is [step], correct?
Let me repeat what I understood.
If I understand correctly, [summary].
Please correct me if I'm wrong.
```

### Ejemplo

> Just to confirm, I should create a new branch from main, implement the change, run the tests locally and open a pull request when it's ready. Is that correct?

## Cómo decir que no entendiste una explicación técnica

### Fórmula

```
I understand the general idea, but I'm not clear on [specific part].
```

### Ejemplo

> I understand the general idea of the deployment process, but I'm not clear on how the staging environment gets updated. Is that triggered manually or through CI/CD?

### Otro ejemplo

> I understand the architecture at a high level, but I'm not clear on how the frontend communicates with the payment service. Could you walk me through that flow?

## Cómo hablar con tu manager

Tu manager es clave durante las primeras semanas. No esperes que adivine tus dudas.

### En tu primera reunión uno a uno

```
What are the priorities for my first week?
What should I focus on first?
What does success look like in the first 30 days?
Are there any documents I should read?
Who should I talk to about [topic]?
How often should we have one-on-one meetings?
```

### Para pedir feedback temprano

```
Do you have any feedback on how I'm doing so far?
Is there anything I should adjust?
Am I moving at the right pace?
Are there any expectations I should be aware of?
```

## Primeras semanas: cómo comunicarte bien

### Si estás aprendiendo

```
I'm still ramping up on the codebase.
I'm getting familiar with the architecture.
I'm reading through the documentation.
I'm pairing with [Name] to understand the flow.
```

### Si vas a tardar

```
This is taking longer than expected because I'm still learning the codebase.
I want to make sure I understand it correctly before making changes.
I'll keep you updated on my progress.
```

### Si encontraste un problema

```
I found an issue in the setup documentation.
The documentation mentions an old command.
The local environment is missing a required package.
I can update the documentation if that would be helpful.
```

Escribí mensajes que respondan:

```
Contexto
Problema
Qué intentaste
Qué necesitás
```

### Mal mensaje

```

```


## Acentos diferentes

En empresas estadounidenses vas a escuchar acentos de muchas partes: Estados Unidos, India, Latinoamérica, Europa, Asia.

Al principio puede ser difícil. Es normal.

### Frases útiles

```
Sorry, the audio cut out for a second.
Could you repeat the last sentence?
I want to make sure I heard that correctly.
Could you type that in the chat?
Let me check if I understood: [repeat summary].
```

### Consejo práctico

No digas:

```
I don't understand your accent.
```

Decí:

```
Sorry, I missed part of that. Could you repeat it?
```

Es más respetuoso.

## Primer ticket o primera tarea

Cuando recibís tu primera tarea, confirmá alcance.

### Frases útiles

```
Could you clarify the expected outcome?
Is there an example of similar work?
What is the priority of this task?
When is this needed?
Are there any edge cases I should consider?
Should I focus on a quick fix or a more complete solution?
```

### Mensaje ejemplo

```
Thanks for assigning this ticket. Before I start, I want to confirm the expected outcome. Should this change only affect the staging environment, or should it also be prepared for production? Also, is there a target date for this?
```

## Cómo recibir feedback

En Estados Unidos, el feedback suele ser directo pero profesional. No lo tomes como ataque personal.

### Frases para aceptar feedback

```
Thanks for the feedback.
That makes sense.
I'll update it.
Good point. I hadn't considered that.
I'll make that change.
Thanks for catching that.
```

### Si no entendés el feedback

```
Could you clarify what you'd like me to change?
Do you mean I should refactor this function or add tests?
Can you share an example of what you have in mind?
```

## Cómo comunicar un blocker

No esperes horas si estás bloqueado. Comunicá temprano.

### Fórmula

```
I'm blocked on [issue].
I tried [actions].
I need [help/access/decision].
```

### Ejemplo

> I'm blocked on deploying to staging. I tried following the deployment guide, but my account doesn't have permission to access the pipeline. Could someone check my permissions or point me to the right person?

## Errores comunes en onboarding

### Error 1: No hacer preguntas

Pensar "no quiero molestar" puede hacer que pierdas días.

Mejor:

```
I have a quick question about the setup process.
Could you clarify this step?
```

### Error 2: No tomar notas

Si preguntás lo mismo dos veces, puede verse mal.

Mejor:

```
I wrote down the steps and I'll follow the documentation first.
```

### Error 3: No comunicar bloqueos

Esperar demasiado sin avisar genera desconfianza.

Mejor:

```
I'm blocked and here's what I tried.
```

### Error 4: Fingir que entendiste

Después se nota.

Mejor:

```
Let me repeat what I understood to make sure I got it right.
```

## Recordatorio final

El onboarding no mide cuánto sabés el primer día. Mide cómo aprendés, cómo comunicás y cómo te integrás.

Usá frases simples, tomá notas, preguntá temprano y confirmá entendimiento.

La frase más importante para recordar es:

> I want to make sure I understand this correctly.

Esa frase te protege de errores, muestra profesionalismo y ayuda a que tu equipo confíe en vos desde el primer día.
