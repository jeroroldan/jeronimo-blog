## URL Structure
- **Theme Landing Pages**: `/blog/{theme}/`
- **Individual Posts**: Keep existing structure `/blog/[slug]/` for backward compatibility, but implement redirects from theme-based URLs if desired
- **Filtered Views**: `/blog?theme={theme}&code={code}&q={search}`

## Navigation Flow
1. **Home Page** → Links to all theme sections andAbout
2. **Theme Landing Pages** → Show all posts in that theme with filtering by code
3. **Individual Post Pages** → Breadcrumb navigation: Home > Theme > Post Title
4. **All Pages** → Access to site-wide search and persistent header/footer

## Theme Mapping Examples
Based on analysis of existing `code` frontmatter:
- **Seducción**: seduccion, atraccion, relaciones-humanas, mindset
- **Informática**: IA, programación, software, desarrollo web, react, laravel, python, etc.
- **Trading**: trading, finanzas, bolsa, inversiones
- **Negocios**: negocios, empresa, emprendimiento, liderazgo, ventas
- **Superación Personal**: superacion, aprendizaje, crecimiento, productividad, gimnasio

## SEO Considerations
- Each theme page is indexable and has unique title/meta description
- Breadcrumb schema implemented on all pages
- Canonical URLs to prevent duplicate content
- XML sitemap automatically generated by `@astrojs/sitemap`

## Implementation Notes
- Theme pages created as `src/pages/blog/[theme].astro`
- Existing `src/pages/blog/[...page].astro` handles default listing and filtering
- Redirects can be implemented via `_middleware.js` or netlify/_redirects if changing URL structure
